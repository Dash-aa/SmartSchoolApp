# Архітектура проєкту SmartSchool

## Загальна структура

Проєкт SmartSchool побудований за принципом багатошарової архітектури. Кожен шар виконує окрему роль, що спрощує підтримку, тестування та розширення системи.

Основні пакети проєкту:
- `controller` — обробка HTTP-запитів і взаємодія з веб-інтерфейсом;
- `service` — реалізація бізнес-логіки;
- `repository` — доступ до даних;
- `dto` — об'єкти для передачі даних із форм;
- `model` — доменні моделі системи.

## Архітектурні рішення

### Використання шаблону MVC

У проєкті використовується підхід MVC:
- **Model** — класи `User`, `Announcement`, `Role`;
- **View** — HTML-шаблони;
- **Controller** — `MainController`, який приймає запити й повертає сторінки.

Такий підхід дозволяє відокремити логіку обробки запитів від відображення даних.

### Винесення бізнес-логіки в service

Логіка авторизації, реєстрації та отримання оголошень не розміщується безпосередньо в контролері. Для цього створені окремі сервіси:
- `AuthService`
- `AnnouncementService`

Це рішення спрощує тестування, повторне використання коду та підтримку проєкту.

### Відокремлення доступу до даних

Для роботи з користувачами використовується інтерфейс `UserRepository` та його реалізація `InMemoryUserRepository`.

Таке рішення дозволяє:
- відокремити бізнес-логіку від способу зберігання даних;
- у майбутньому замінити in-memory сховище на базу даних без зміни сервісів.

### Використання DTO для форм

Для обробки введених користувачем даних використовуються:
- `LoginForm`
- `RegisterForm`

Це дозволяє не передавати напряму доменні моделі у форми, а також окремо налаштовувати валідацію вхідних даних.

## Переваги обраної архітектури

- зрозумілий розподіл відповідальності між компонентами;
- простіше тестувати окремі частини системи;
- легше супроводжувати та розширювати код;
- зручніше документувати проєкт.
