# Оновлення системи (Production)

## 1. Підготовка до оновлення

Перед початком оновлення необхідно:

* перевірити стабільність поточної версії
* переконатися, що є доступ до сервера
* попередити користувачів про можливий downtime
* перевірити доступність необхідних ресурсів

---

## 2. Резервне копіювання

У поточній версії (in-memory):

* резервне копіювання не застосовується

У production (з БД):

* виконати backup бази даних
* зберегти конфігураційні файли

Приклад:

pg_dump smartschool > backup.sql

---

## 3. Перевірка сумісності

* перевірити зміни у залежностях
* перевірити сумісність версій Java
* перевірити конфігурації

---

## 4. Планування простою (Downtime)

* визначити час оновлення (наприклад, ніч)
* мінімізувати вплив на користувачів

---

## 5. Процес оновлення

### 5.1 Зупинка сервісу

pkill -f smartschool

---

### 5.2 Отримання нового коду

git pull

---

### 5.3 Збірка

mvn clean package

---

### 5.4 Розгортання

java -jar target/*.jar

---

### 5.5 Міграція даних

У поточній версії не використовується.

У production:

* виконати SQL-міграції (якщо є)

---

### 5.6 Оновлення конфігурацій

* перевірити application.properties
* оновити змінні середовища (якщо потрібно)

---

## 6. Перевірка після оновлення

Переконатися, що:

* застосунок запускається без помилок
* сторінки відкриваються
* авторизація працює
* немає критичних помилок у логах

---

## 7. Rollback (відкат)

У разі помилки:

git checkout HEAD~1
mvn clean package
java -jar target/*.jar

Або:

* відновити backup
* повернути попередню версію

---

## 8. Висновок

Оновлення повинно виконуватися контрольовано з попереднім backup і перевіркою після запуску.
