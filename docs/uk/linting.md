# Linting у проєкті SmartSchool

## Обраний лінтер та причини вибору

У проєкті SmartSchool використано інструмент статичного аналізу коду **Checkstyle**.

Checkstyle було обрано, оскільки він:
- добре підходить для Java-проєктів;
- інтегрується з Maven;
- дозволяє забезпечити єдиний стиль коду;
- допомагає знаходити типові помилки;
- покращує читабельність і підтримуваність коду.

Додатково у проєкті використовуються:
- **SpotBugs** — для пошуку потенційних помилок у коді;
- **Spotless** — для автоматичного форматування коду.

---

## Базові правила та їх пояснення

У межах лабораторної роботи було створено конфігураційний файл **checkstyle.xml**.

Основні правила:

- **UnusedImports** — заборона невикористаних імпортів;
- **AvoidStarImport** — заборона використання `*` в імпортах;
- **NeedBraces** — обов’язкове використання фігурних дужок;
- **WhitespaceAfter / NoWhitespaceBefore** — контроль пробілів;
- **RedundantImport** — заборона дублювання імпортів;
- **EmptyBlock** — заборона порожніх блоків;
- **EqualsHashCode** — узгодженість методів equals() і hashCode();
- **HiddenField** — заборона приховування полів параметрами;
- **LineLength** — обмеження довжини рядка (120 символів);
- **FileTabCharacter** — заборона табуляції.

Ці правила забезпечують єдиний стиль і якість коду.

## Ігноровані директорії

Під час перевірки ігноруються:
- `target/` — згенеровані файли;
- `.idea/` — службові файли середовища розробки.

---

## Інструкція з запуску лінтера

Основна команда:

mvn checkstyle:check

Генерація звіту:

mvn checkstyle:checkstyle

Повна перевірка:

mvn validate

---

## Результати перевірки

Після запуску лінтера отримано:

You have 0 Checkstyle violations.  
BUILD SUCCESS

Це означає, що код відповідає всім заданим правилам.

## Git hooks

Було налаштовано pre-commit hook для автоматичної перевірки коду перед комітом.

Перед кожним комітом запускаються:
- checkstyle
- spotless
- spotbugs
- тести

Якщо виникає помилка — коміт скасовується.

Файл hook знаходиться у:

.git/hooks/pre-commit

Приклад:

#!/bin/sh
mvn checkstyle:check spotless:check spotbugs:check test
if [ $? -ne 0 ]; then
echo "Checks failed. Commit aborted."
exit 1
fi

Примітка: папка `.git` не зберігається в репозиторії, тому hook описано в документації.

## Інтеграція з процесом збірки

Лінтинг інтегровано в Maven lifecycle:

- Checkstyle — фаза validate
- Spotless — фаза validate
- SpotBugs — фаза verify

Команда:

mvn clean verify

Під час виконання автоматично відбувається:
- перевірка стилю;
- перевірка форматування;
- компіляція;
- запуск тестів;
- аналіз коду.

---

## Статична типізація

Java є строго типізованою мовою, тому окремі інструменти не потрібні.

Додатково використано параметр компілятора:

-Xlint:all

Це дозволяє виявляти:
- небезпечні конструкції;
- помилки типізації;
- застарілий код;
- інші потенційні проблеми.

---

## Комплексна перевірка коду

Для повної перевірки використовується:

mvn clean verify

Або:

mvn clean validate test spotbugs:check spotless:check
