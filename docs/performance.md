# Performance profiling for SmartSchool

## Methodology

Для аналізу продуктивності застосунку SmartSchool було використано такі інструменти:

- Spring Boot Actuator для збору метрик HTTP-запитів
- Вбудовані логи Spring Boot для визначення часу запуску
- Інструменти браузера (DevTools → Network → Timing) для вимірювання часу обробки запитів

Було визначено тестові сценарії:
1. Запуск застосунку
2. Відкриття головної сторінки (/)
3. Вхід користувача (/login)
4. Реєстрація користувача (/register)

---

## Baseline metrics

Отримані наступні базові метрики продуктивності:

- Запуск застосунку: 3.69 с
- Головна сторінка: ~120 мс
- Логін: ~100 мс
- Реєстрація: ~150 мс
- Використання пам’яті (Heap): ~120 MB

---

## Hot spots

За результатами профілювання було визначено наступні "гарячі точки":

1. **AuthService.register()**
    - Найбільший час виконання (~150 мс)
    - Включає валідацію, перевірки та створення нового користувача

2. **AnnouncementService.getActualAnnouncements()**
    - Викликається при відкритті головної сторінки (~120 мс)
    - Впливає на швидкість відображення домашньої сторінки

3. **AuthService.login()**
    - Виконується при кожному вході (~100 мс)
    - Включає пошук користувача та перевірку пароля

---

## Identified performance issues

Під час аналізу було виявлено наступні проблеми:

- Повторне створення об’єктів оголошень при кожному запиті
- Відсутність кешування результатів для головної сторінки
- Лінійний пошук користувача (у разі використання списку)
- Зайві перевірки під час реєстрації

---

## Conclusion

Застосунок SmartSchool демонструє стабільну та достатньо швидку роботу, середній час обробки запитів становить близько 100–150 мс.

Основні вузькі місця пов’язані з операціями реєстрації, завантаження оголошень та авторизації користувачів.

Подальша оптимізація цих ділянок дозволить покращити загальну продуктивність системи.
