# Test-Driven Documentation

У проєкті SmartSchool тести використовуються не лише для перевірки коректності роботи, але і як приклади використання компонентів системи.

## AuthService

Тести для AuthService демонструють:

- як виконується авторизація користувача;
- що відбувається при неправильному паролі;
- як працює реєстрація користувача;
- які помилки повертаються при некоректних даних.

Таким чином, тестові методи показують, як викликати методи `login()` та `register()` і які результати очікувати.

## AnnouncementService

Тести для AnnouncementService демонструють:

- як отримати список актуальних оголошень;
- який формат даних повертає сервіс;
- як працює метод `getActualAnnouncements()`.

## Переваги підходу

Використання тестів як документації має такі переваги:

- тести завжди актуальні разом із кодом;
- показують реальні приклади використання;
- допомагають новим розробникам швидше зрозуміти систему;
- дозволяють перевіряти правильність роботи системи автоматично.

Таким чином, тести виступають як "жива" документація, що відображає реальну поведінку системи.
